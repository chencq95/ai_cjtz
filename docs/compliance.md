# 公开元数据采集、安全与合规边界

## 1. 原则与责任

本项目只采集经核验允许自动化访问的公开元数据，不以“网页可见”为充分授权依据。robots、用户协议、版权声明、接口条款、平台通知和双方授权需综合评估。

本文件是工程控制基线，不替代法律意见。平台从 `PENDING_AUDIT` 进入 `ACTIVE` 前，业务负责人、数据负责人和合规负责人应共同确认范围并留下审计记录。

基本原则：

- 合法、正当、必要、诚信。
- 目的明确和数据最小化。
- 不绕过登录、验证码、付费、接口签名或访问控制。
- 不使用代理轮换、指纹伪造等手段规避平台限制。
- 不采集与产品目录分析无关的个人信息和交易明细。
- 原始内容、结构化事实和 Agent 可见数据分层授权。
- 每个字段可追溯到来源、抓取时间和解析证据。

需要纳入评审的主要法规和规则包括但不限于：

- 《中华人民共和国网络安全法》
- 《中华人民共和国数据安全法》
- 《中华人民共和国个人信息保护法》
- 《网络数据安全管理条例》
- 平台自身用户协议、隐私政策、版权声明、API 条款和 robots 策略

法规和平台条款可能变化，不能把本文件中的当前判断视为永久授权。

## 2. 公开元数据范围

### 2.1 默认可纳入候选范围

满足“无需登录、公开展示、与目录检索直接相关、未被条款禁止”时，可进入合规评审：

- 产品、组件、场景、需求、公开数商的名称和公开编号。
- 平台原生分类、产品形态、行业、地域和标签。
- 公开摘要、功能说明、应用场景和交付方式。
- 提供机构、运营机构等组织信息。
- 公开发布时间、更新时间、数据覆盖周期、挂牌状态。
- 公开价格表达，例如固定价、区间、面议；不推断未公开成交价。
- 官网公开的 API 说明、样例字段和更新频率。
- 公开详情 URL、来源平台和页面证据。

### 2.2 条件采集

以下内容必须有额外用途、授权和保留策略：

- 可下载的数据样例、PDF、Office、压缩包和接口文档。
- 大段产品说明、研究报告、案例全文和受版权保护图片。
- 个人姓名、手机号、邮箱、微信、身份证件或其他可识别个人的信息。
- 需要接受额外协议才能下载的资料。
- 公开页面中疑似包含敏感数据、重要数据或不应公开的信息。
- 来源站点公开页面调用但声明仅供前端使用的接口。

条件采集项默认只在原始隔离层保留，结构化层和 Agent 层是否暴露由字段级策略决定。

### 2.3 默认禁止

- 登录后、会员、付费或特定机构可见内容。
- 验证码后内容、反自动化保护后的内容。
- 订单、合同、结算、交易对手、成交明细等非公开业务数据。
- 账号、Cookie、Authorization、签名密钥和内部接口凭据。
- 来源页面意外暴露但明显不属于公开目录的个人信息或敏感数据。
- 通过漏洞、参数遍历、未授权对象引用或接口逆向获得的数据。
- 未经授权对 robots 或用户协议明确禁止的路径进行自动化采集。

发现疑似泄露、敏感信息或错误公开时，应立即停止扩散，隔离快照并通知合规负责人，不得因技术上可访问就继续批量采集。

## 3. 每个平台的合规登记

`source_platform` 和审计表至少记录：

- 平台主体、官方域名和核验依据。
- `robots.txt` URL、获取时间、原文哈希和解析结果。
- 用户协议、隐私政策、版权/API 条款 URL、获取时间和快照。
- 允许和禁止的路径、对象类型、文件类型。
- 是否允许公开 API、静态抓取、浏览器渲染和附件下载。
- 每域名速率、并发、允许时段和识别性 User-Agent。
- 处理目的、数据类别、保留期限和 Agent 暴露级别。
- 合规决定、决定人、决定时间、复核日期和附加条件。
- 官方授权、邮件或合同的受控引用，不在业务库保存敏感凭据。

`robots.txt` 不是唯一法律判断，但工程侧必须至少同等严格执行。robots 获取失败时不得默认“全部允许”；新来源保持 `PENDING_AUDIT`，由人工确认后再启用。

条款、robots 或域名发生变化时，自动将对应 collection 置为 `REVIEW_REQUIRED`。复核完成前只允许保守健康探测，不开展扩范围抓取。

## 4. 访问行为规范

### 4.1 身份和限速

- 使用固定、可识别的 User-Agent，并提供项目或运维联系渠道。
- 每域名独立令牌桶和并发上限，默认从低速开始。
- 尊重 `Retry-After`、robots crawl-delay 及平台明确要求。
- 429、403、验证码或风控页出现后降低速率或熔断，不更换 IP绕过。
- 每日任务在平台间加入抖动，避免 02:30 同时请求全部官网。

### 4.2 动态页面和公开 API

Playwright 只用于渲染公开页面和观察该页面正常发起的网络请求。某接口被浏览器调用不自动等于可长期直接调用；固化为 API 适配器前必须核验：

- 无需登录、Cookie、临时 token 或用户身份。
- 没有破解签名、混淆或加密的行为。
- 参数范围与公开页面实际能力一致。
- 使用方式不违反条款或平台明确限制。
- 请求速率不高于正常公开页面访问的合理水平。

浏览器上下文使用临时配置，不加载个人浏览器资料、扩展、Cookie 或保存的密码。

## 5. SSRF 与网络出口控制

来源 URL、重定向、页面链接和接口返回 URL 都是不可信输入。抓取器必须实施：

### 5.1 域名和协议白名单

- 只允许注册表审批过的 `https`/必要 `http` 域名和端口。
- 默认拒绝 `file:`、`ftp:`、`gopher:`、`data:`、`javascript:` 等协议。
- 新子域、CDN、附件域和重定向目标必须先审批再加入白名单。
- 禁止用户或 Agent 临时提交任意 URL让生产抓取器访问。

### 5.2 地址检查

DNS 解析前后都要检查并阻断：

- 回环地址：`127.0.0.0/8`、`::1`。
- 私网和链路本地地址。
- 云元数据地址和主机管理网段。
- IPv4-mapped IPv6、十进制/八进制/十六进制混淆地址。
- DNS rebinding：连接时使用已核验解析结果，并对每次重定向重新校验。

生产部署应通过受控出口代理或防火墙只允许审批域名，应用层白名单不能替代网络层隔离。

### 5.3 重定向和下载

- 限制重定向次数，逐跳记录和校验。
- 限制响应大小、解压后大小、超时和下载速率。
- 校验声明 Content-Type、文件魔数和扩展名的一致性。
- 压缩包防止 Zip Bomb、路径穿越和递归嵌套。
- 文件只进入隔离对象存储，不在抓取主机上打开或执行。

命中 SSRF 或非白名单目标时记录 `SECURITY_BLOCK`，不自动重试。

## 6. 文件和内容安全

- PDF、Office、图片和压缩包保存前计算 SHA-256。
- 使用恶意软件扫描器；未扫描或命中样本不得进入解析和 Agent 层。
- 文档解析器在无网络、低权限、资源受限的隔离进程/容器中运行。
- 禁止宏、脚本、嵌入对象和外部链接自动执行。
- OCR 和文档解析结果仍视为不可信文本。
- 原始文件与解析文本分别记录对象 URI、哈希、解析器版本和状态。

## 7. 个人信息与敏感内容

### 7.1 最小化

目录检索通常只需要机构级信息。个人联系人不属于默认核心字段：

- 页面公开手机号、邮箱、微信等默认不进入标准实体和 Agent 检索索引。
- 如业务确需联系方式，先记录明确用途、合法处理依据、访问角色和保留期限。
- UI/API 默认脱敏，原文只允许受控角色查看。
- 禁止用爬取信息构建个人画像或跨站关联个人身份。

### 7.2 检测和隔离

解析管道在标准化前执行 PII/敏感内容检测，包括手机号、邮箱、证件号、账号、精确地址和疑似密钥。命中后：

1. 原始快照标记 `SENSITIVE_REVIEW`。
2. 默认不进入 Agent 索引。
3. 输出字段按规则删除、泛化或脱敏。
4. 高风险内容进入人工复核。
5. 记录检测器版本、命中类型和处置，不在普通日志复制原值。

### 7.3 主体权利和删除

建立更正、删除、限制使用和来源申诉流程。即使来源网页仍存在，合规决定要求停止提供时，Agent 服务应能立即隐藏对应字段；原始审计副本的保留按法律和安全要求单独决定。

## 8. Agent 与提示注入防护

公开网站可能包含故意或无意的指令文本。所有爬取内容必须被标记为“不可信数据”，不能改变 Agent 指令层级。

### 8.1 数据分层

- `RAW`：完整响应，仅审计和重放角色可访问。
- `NORMALIZED`：经过解析、类型校验、PII 和安全处理的结构化事实。
- `AGENT_SAFE`：面向 REST/MCP 的字段白名单、短证据片段和来源链接。

Agent 只能默认访问 `AGENT_SAFE`。长正文需要专门工具和权限，返回时明确标记引用边界。

### 8.2 工具边界

- MCP/REST 只读、参数化查询，不提供任意 SQL。
- 不允许 Agent 指定任意 URL、文件路径、命令或适配器代码。
- 工具对返回数量、字段、时间范围和文本长度设上限。
- Agent 不得根据页面文本发送邮件、上传文件、调用外部 URL或修改调度。
- 来源文本中的“忽略规则”“调用工具”“泄露密钥”等内容只能作为数据引用，不能执行。

### 8.3 检测和审计

- 对正文和 OCR 文本运行提示注入模式检测，命中不等于删除原始证据，但会降低 Agent 暴露级别。
- 保留查询参数、返回实体 ID、版本、来源、策略决定和工具调用审计。
- 返回值包含 `source_url`、`data_as_of`、`last_crawled_at` 和字段置信度。
- 模型分类输出不能覆盖原始事实，也不能在无证据时补造产品属性。

## 9. 数据版权、再发布和引用

- 数据库内部可为审计保存获准原始页面快照，但 Agent 默认提供结构化事实、有限摘要和来源链接。
- 不默认对外再发布整页文字、完整报告、图片或附件。
- 来源站点要求署名时，在 API/UI 中保留平台名称和原始 URL。
- 来源撤回、纠错或下架后保留版本历史的权限需与审计必要性、合同和保留政策共同评估。
- 跨平台聚合结果不能暗示本项目拥有来源数据产品本身的权利。

## 10. 分类分级、加密和访问控制

建议分级：

- `PUBLIC_METADATA`：经审核可向 Agent 返回的公开目录字段。
- `INTERNAL_DERIVED`：分类、相似度、质量分和运营统计。
- `RESTRICTED_RAW`：原始 HTML/JSON、附件、截图和可能含 PII 的内容。
- `SECURITY_QUARANTINE`：恶意文件、疑似泄露、提示注入或高风险内容。

控制要求：

- 传输使用 TLS；数据库、对象存储和备份静态加密。
- 服务账号最小权限，抓取、解析、发布和 Agent 查询角色分离。
- 原始层和隔离区禁止普通 Agent/API 账号读取。
- 管理操作启用强认证、审计和定期权限复核。
- 凭据来自受控密钥系统，不写入仓库、日志、任务参数或截图。

## 11. 保留、删除和备份

保留期不应在代码中永久硬编码。每类数据在策略表中记录：

- 处理目的和负责人。
- 法律/合同依据。
- 原始快照、结构化当前值、历史版本、日志和备份的保留期。
- 到期动作：删除、匿名化或转存审计档案。
- 法律保全或安全事件冻结条件。

删除需覆盖 PostgreSQL、对象存储、搜索索引、向量、缓存和后续备份过期流程。删除请求和执行结果保留不可篡改审计，但审计本身不得重复保存被删除的敏感原值。

## 12. 事件响应

### 12.1 触发事件

- 发现非公开、敏感或个人信息被错误采集。
- SSRF、恶意文件、凭据或提示注入命中。
- 平台投诉、授权撤回或条款变化。
- 未授权访问、数据泄露、异常批量导出。
- 抓取频率失控，对来源造成影响。

### 12.2 处置步骤

1. 熔断相关平台、collection 或 Agent 字段，不扩大影响。
2. 隔离原始对象和已发布索引，保留必要审计证据。
3. 确认影响来源、时间、字段、版本、查询和下游。
4. 通知安全、合规、业务负责人及必要外部主体。
5. 删除、脱敏、更正或恢复访问，并完成下游同步。
6. 通过 replay 和回归测试验证修复。
7. 记录根因、时间线、决定、修复和预防措施。

事件响应不得通过销毁必要审计证据掩盖问题，也不得在普通工单中粘贴敏感原文。

## 13. 上线检查清单

平台启用前必须逐项通过：

- [ ] 主体、官方域名、跳转和附件域已核验。
- [ ] robots、条款、隐私和版权/API 边界有快照和结论。
- [ ] 公开元数据字段清单和明确排除项已批准。
- [ ] 请求速率、并发、User-Agent 和重试策略已批准。
- [ ] 没有登录、验证码、签名破解或代理规避逻辑。
- [ ] SSRF、DNS rebinding、重定向和下载大小防护通过测试。
- [ ] 文件隔离、魔数校验和恶意软件扫描生效。
- [ ] PII 检测、脱敏、隔离和主体权利流程通过测试。
- [ ] Agent 只读字段白名单和提示注入防护通过测试。
- [ ] 原始、规范、Agent-safe 三层权限隔离通过测试。
- [ ] 凭据未出现在仓库、日志、任务参数和数据库明文字段中。
- [ ] 保留、删除、备份和事件响应负责人已确定。
- [ ] 合规复核日期和触发条件已设置。

任何未通过项都必须有明确阻断状态或书面风险接受，不能用“后续补充”替代上线门禁。
